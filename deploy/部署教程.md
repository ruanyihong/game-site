# 课件0 部署到 GitHub Pages · 图文教程（电脑小白版）

目标：把这份网页课件变成一个谁都能打开的网址，随时发链接分享。

> 你只要「点几下鼠标」，**完全不用敲命令行**。预计 10 分钟搞定。

---

## 第一步：装一个「GitHub 桌面版」软件（只装一次）

1. 打开浏览器，访问 **https://desktop.github.com**
2. 点大大的 **Download for macOS** 按钮，下载一个 `.dmg` 文件。
3. 下载完后双击它 → 把里面的 **GitHub Desktop** 图标拖到「应用程序」文件夹。
4. 从「启动台」或「应用程序」里打开 **GitHub Desktop**。

---

## 第二步：登录你的 GitHub 账号

1. 打开后会出现欢迎界面，点 **Sign in to GitHub.com**（用 GitHub 登录）。
2. 浏览器会自动打开，输入你的 **GitHub 用户名 + 密码**（如果有验证码/两步验证，按提示完成）。
3. 登录成功后回到 GitHub Desktop，可能会让你填名字和邮箱——直接填你的常用信息即可，然后点 **Finish**（完成）。

---

## 第三步：把课件文件夹交给 GitHub 桌面版

1. 在 GitHub Desktop 顶部菜单点 **File（文件）→ Add Local Repository…（添加本地仓库）**
2. 点 **Choose…（选择）**，找到这个文件夹：
   ```
   /Users/yihong/WorkBuddy/2026-07-14-13-25-31/wb-kejian01/ppt-kejian0/deploy
   ```
   （里面应该有 `index.html`、`assets` 文件夹、`.nojekyll` 这几个东西）
3. 它会提示「这个文件夹还不是仓库」，点 **Create a Repository（创建仓库）**，本地路径就是上面的 deploy 文件夹，其他默认，点确认。

---

## 第四步：提交（相当于「保存」）

1. 左下角有一个填写框，写一句说明，比如：`课件0 初版发布`
2. 点下方的 **Commit to main**（提交到 main 分支）。

---

## 第五步：发布到网上（Publish）

1. 点窗口右上角的 **Publish repository**（发布仓库）。
2. **重要**：把 **Keep this code private（设为私有）** 的勾 **取消掉**（必须公开，否则免费版无法生成网页）。
3. 仓库名建议填：`wb-kejian0`（这就是你网址里的一部分）。
4. 点 **Publish（发布）**，等待进度条走完（首次可能要十几秒）。

---

## 第六步：打开「网页托管」开关

1. 打开浏览器，访问 **https://github.com** 并登录。
2. 点右上角你的头像 → **Your repositories（你的仓库）** → 进入刚发布的 **wb-kejian0**。
3. 点右上方的 **Settings（设置）**。
4. 在左侧菜单找到 **Pages**（页面）。
5. 在 **Build and deployment（构建与部署）** 里：
   - Source（源）选 **Deploy from a branch（从分支部署）**
   - Branch（分支）选 **main**，文件夹选 **/ (root)**（根目录）
   - 点 **Save（保存）**。

---

## 第七步：拿到你的分享链接 🎉

1. 保存后页面会刷新，稍等 **1～2 分钟**（GitHub 第一次生成网页需要一点时间）。
2. 在 Pages 页面上方会出现一行绿字，类似：
   ```
   Your site is live at https://你的用户名.github.io/wb-kejian0/
   ```
3. 把这条链接复制发给任何人，他们用手机/电脑浏览器打开就能看课件了！

> 小提示：以后如果课件有更新，把新文件放进 `deploy` 文件夹，GitHub Desktop 会自动发现变化，写句说明点 **Commit & Push** 就行，网址不变。

---

## 备选方案：不想装软件？用网页直接传（免安装）

如果你不想装 GitHub Desktop，也可以纯网页操作：

1. 浏览器访问 **https://github.com** → 点右上角 **+** → **New repository（新建仓库）**。
2. 仓库名填 `wb-kejian0`，选 **Public（公开）**，直接点 **Create repository（创建）**。
3. 创建后页面会让你「上传文件」：把 `deploy` 文件夹里的 **所有内容**（`index.html`、`assets` 文件夹、`.nojekyll`）一起拖进上传区。
4. 拖完后拉到最下面点 **Commit changes（提交）**。
5. 然后走上面「第六步」打开 Pages 开关即可。

> 注意：网页上传对包含子文件夹的情况偶尔会卡，如果传不上去，还是装 GitHub Desktop 最稳。

---

## 常见问题

- **打开是空白/乱码？** 确认 `assets` 文件夹和 `index.html` 是放在一起上传的，且仓库是 **Public**。
- **图片不显示？** 检查 `assets` 文件夹是否一起上传了（最容易漏掉）。
- **网址打不开？** 首次发布后等 1～2 分钟再试；若仍不行，回到 Settings → Pages 确认分支是 `main`、文件夹是 `/ (root)`。
- **字体变普通了？** 课件用了 Google 字体中国镜像，需要联网；断网时会自动用系统字体，不影响阅读。
